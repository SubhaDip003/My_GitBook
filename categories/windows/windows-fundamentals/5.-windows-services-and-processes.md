# 5. Windows Services & Processes

## Windows Services <a href="#windows-services" id="windows-services"></a>

Windows Services are background processes in Windows that run continuously and perform important system tasks like networking, updates, authentication, and system monitoring. These services can start automatically when the system boots and continue running even if no user is logged in. Many applications can also install their own services, especially in server environments. All services are managed by the **Service Control Manager (SCM)**, and you can view and manage them using the graphical tool **`services.msc`**, which shows details like service name, status (Running, Stopped, Paused), startup type (Automatic, Manual, Delayed), and the account the service runs under (Local System, Network Service, etc.).

From the command line or PowerShell, services can be queried and managed easily. The command lists all services:

```powershell
Get-Service
```

shows only running services:

```powershell
Get-Service | ? {$_.Status -eq "Running"}
```

You can also format output for better readability using:

```powershell
Get-Service | select -First 2 | fl
```

which displays detailed properties like name, status, dependencies, and service type.

Using the traditional command-line tool:

```cmd
sc query
```

you can check service status, and can start or stop services.

```cmd
sc start <service_name>
sc stop <service_name>
```

Services run under different privilege levels such as **Local System (highest privileges), Network Service, and Local Service**, and usually only administrators can create or modify them. Because of this, misconfigured service permissions are commonly used in privilege escalation attacks. Some critical services cannot be stopped without restarting the system, and if files used by those services are modified, a system reboot is required for changes to take effect.

Some critical system services are:

| Service                   | Description                                                                                                                                                                                                                                             |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| smss.exe                  | Session Manager SubSystem. Responsible for handling sessions on the system.                                                                                                                                                                             |

| wininit.exe               | Starts the Wininit file .ini file that lists all of the changes to be made to Windows when the computer is restarted after installing a program.                                                                                                        |

## Processes <a href="#processes" id="processes"></a>

Processes in Windows are programs or tasks that run in the background. Some processes start automatically as part of the operating system, while others are started when you open applications. Every application you run (like a browser or tool) creates one or more processes, and Windows itself also runs many internal processes to keep the system working.

Some processes are normal and can be safely stopped without affecting the system too much, especially those related to user-installed applications. However, there are also **critical system processes** that are essential for Windows to function properly. If these are stopped, the system may crash, freeze, or restart. Examples include processes like the Windows Logon Application, System process, System Idle Process, Client Server Runtime, Windows Session Manager, Service Host (svchost), and the Local Security Authority Subsystem Service (LSASS), which is responsible for handling authentication and security policies.

***

## Local Security Authority Subsystem Service (LSASS) <a href="#local-security-authority-subsystem-service-lsass" id="local-security-authority-subsystem-service-lsass"></a>

`lsass.exe` is the process that is responsible for enforcing the security policy on Windows systems. When a user attempts to log on to the system, this process verifies their log on attempt and creates access tokens based on the user's permission levels. LSASS is also responsible for user account password changes. All events associated with this process (logon/logoff attempts, etc.) are logged within the Windows Security Log. LSASS is an extremely high-value target as several tools exist to extract both cleartext and hashed credentials stored in memory by this process.

## Sysinternals Tools <a href="#sysinternals-tools" id="sysinternals-tools"></a>

The [SysInternals Tools suite](https://docs.microsoft.com/en-us/sysinternals) is a set of portable Windows applications that can be used to administer Windows systems (for the most part without requiring installation). The tools can be either downloaded from the Microsoft website or by loading them directly from an internet-accessible file share by typing `\\live.sysinternals.com\tools` into a Windows Explorer window.

For example, we can run procdump.exe directly from this share without downloading it directly to disk.

These tools can be useful for penetration testers to, for example, discover interesting processes and possible privilege escalation paths as well as for lateral movement.

## Task Manager <a href="#task-manager" id="task-manager"></a>

Windows Task Manager is a powerful tool for managing Windows systems. It provides information about running processes, system performance, running services, startup programs, logged-in users/logged in user processes, and services. Task Manager can be opened by right-clicking on the taskbar and selecting `Task Manager`, pressing ctrl + shift + Esc, pressing ctrl + alt + del and selecting `Task Manager`, opening the start menu and typing `Task Manager`, or typing `taskmgr` from a CMD or PowerShell console.

## **Process Explorer**

[**Process Explorer**](https://learn.microsoft.com/en-us/sysinternals/downloads/process-explorer) is a powerful tool from the Sysinternals suite used to view and analyze running processes in Windows. It works like an advanced version of Task Manager but provides much more detailed information. It shows all currently running processes and allows you to see what each process is doing internally.

With Process Explorer, you can view **handles and DLLs** used by a process. Handles are references to system resources (like files or registry keys), and DLLs are libraries loaded by a program. This helps in understanding how a program is interacting with the system. You can also search inside the tool to find which process is using a specific file, DLL, or resource.

Another important feature is the ability to see **parent-child process relationships**. This means you can identify which process started another process. This is very useful for detecting suspicious activity or troubleshooting issues like **orphaned processes** (processes left running even after the main program is closed).

Overall, Process Explorer is widely used for **system monitoring, malware analysis, and troubleshooting**, as it gives deep visibility into how processes behave on a Windows system.
