# 4. NTFS vs. Share Permissions

NTFS permissions and Share permissions are both used in Windows to control access to files and folders, but they work in different ways and in different situations. Windows is widely used across organizations, which makes it a common target for malware. Many attacks spread through network shares, especially when permissions are weak or misconfigured. Technologies like SMB (Server Message Block) are used to share files and printers over a network, allowing users to access resources from other systems.

NTFS permissions are applied directly on the system where the files and folders are stored. These permissions control what a user can do locally or through remote access like RDP. By default, NTFS permissions are inherited from parent folders, but this inheritance can be disabled to set custom permissions. NTFS provides detailed and granular control, such as allowing or denying read, write, or modify access.

Share permissions, on the other hand, are applied when a folder is accessed over the network using SMB. These permissions only come into effect when the resource is accessed from another system. If a user logs in locally to the machine, Share permissions do not apply; only NTFS permissions are checked.

In practice, both NTFS and Share permissions can apply to the same folder when it is shared over the network. The final access a user gets is determined by combining both, and the most restrictive permission will be applied. This makes it important to configure both correctly to avoid unintended access or security risks.

### Share Permissions

| Permission       | Description                                                                                                         |
| ---------------- | ------------------------------------------------------------------------------------------------------------------- |
| **Full Control** | Allows users to do everything: read, create, modify, delete files and folders, and also change permissions on them. |
| **Change**       | Allows users to read, create, edit, and delete files and subfolders, but cannot change permissions.                 |
| **Read**         | Allows users only to view files and folder contents; no editing or deleting is allowed.                             |

### **NTFS Basic permissions**

| Permission               | Description                                                                               |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| **Full Control**         | User can do everything: read, write, modify, delete files/folders, and change permissions |
| **Modify**               | User can read, edit, and delete files/folders but cannot change permissions               |
| **Read & Execute**       | User can open/read files and run programs                                                 |
| **List Folder Contents** | User can see files and folders inside a directory (mainly for folders)                    |
| **Read**                 | User can only view or open files, cannot make changes                                     |
| **Write**                | User can create new files and make changes to existing files                              |
| **Special Permissions**  | Advanced/custom permissions for specific actions (like delete, change ownership, etc.)    |

### **NTFS special permissions**

| Permission                         | Description                                                                                                           |
| ---------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| **Full Control**                   | User can do everything — create, edit, move, and delete files/folders, and also change permissions or take ownership. |
| **Traverse Folder / Execute File** | User can pass through folders even if they cannot open them, and can run executable files (programs).                 |
| **List Folder / Read Data**        | User can see files and folders inside a directory and open/read the files.                                            |
| **Read Attributes**                | User can view basic properties of a file/folder like hidden, read-only, archive, etc.                                 |
| **Read Extended Attributes**       | User can view advanced properties set by applications (depends on software).                                          |
| **Create Files / Write Data**      | User can create new files inside a folder and modify existing file content.                                           |
| **Create Folders / Append Data**   | User can create subfolders and add data to files, but cannot overwrite existing data.                                 |
| **Write Attributes**               | User can change basic file properties (like hidden or read-only), but cannot create files/folders.                    |
| **Write Extended Attributes**      | User can modify advanced attributes set by programs.                                                                  |
| **Delete Subfolders and Files**    | User can delete files and subfolders inside a folder, but not the main parent folder.                                 |
| **Delete**                         | User can delete the selected file/folder (including parent folder if allowed).                                        |
| **Read Permissions**               | User can view what permissions are set on a file or folder.                                                           |
| **Change Permissions**             | User can modify permissions assigned to other users/groups.                                                           |
| **Take Ownership**                 | User can become the owner of a file/folder and then get full control over it.                                         |
