# 10. Microsoft Management Console (MMC)

**Microsoft Management Console (MMC)** is a Windows tool used to manage different system components like hardware, software, and network settings from one place. It acts like a central framework where administrators can load different administrative tools called **snap-ins**.

Snap-ins are individual management tools (for example: Services, Event Viewer, Disk Management) that can be added into MMC. This allows you to create a **custom console** with only the tools you need, instead of opening multiple separate utilities. These snap-ins can be used to manage both the **local system and remote systems**, which is very useful in enterprise environments.

MMC has been available since Windows Server 2000 and is still used in modern Windows systems. It is mainly used by administrators to simplify system management, automate tasks, and organize tools in a single interface.

We can open MMC by just typing `mmc` in the Start menu. When we open MMC for the first time, it will be blank.

From here, we can browse to File --> `Add or Remove Snap-ins`, and begin customizing our administrative console.

As we begin adding snap-ins, we will be asked if we want to add the snap-in to manage just the local computer or if it will be used to manage another computer on the network.

<figure><img src="../../../.gitbook/assets/image (131).png" alt=""><figcaption></figcaption></figure>

Once we have finished adding snap-ins, they will appear on the left-hand side of MMC. From here, we can save the set of snap-ins as a .msc file, so they will all be loaded the next time we open MMC. By default, they are saved in the Windows Administrative Tools directory under the Start menu. Next time we open MMC, we can choose to load any of the views that we have created.
