# 1. Introduction to Windows

The Windows operating system was first introduced by Microsoft on November 20, 1985, initially as a graphical shell for MS-DOS. Over time, it evolved into a fully independent operating system with major releases such as Windows 95, which marked the first true integration of Windows and DOS and introduced built-in internet support along with Internet Explorer. Microsoft continued developing newer versions including Windows XP, Vista, 7, 8, 10, and the latest Windows 11, offering multiple editions tailored for home users, professionals, and enterprise environments.

On the server side, Microsoft introduced Windows Server with Windows NT 3.1 Advanced Server in 1993. Over the years, it incorporated key technologies like Internet Information Services (IIS), Active Directory (introduced in Windows 2000), Microsoft Management Console (MMC), and advanced networking features. Later versions such as Windows Server 2003, 2008, 2012, 2016, and 2019 introduced enhancements like built-in firewalls, Hyper-V virtualization, failover clustering, and support for modern technologies like Kubernetes and Linux containers.

As newer versions are released, older ones eventually reach end-of-life and stop receiving security updates. However, some legacy systems are still used in organizations due to compatibility or cost constraints, which can introduce security risks. For example, systems like Windows Server 2008 and 2012 are no longer supported, although emergency patches have been released in critical cases such as the SMBv1 EternalBlue vulnerability.

Windows versions can be identified by their internal version numbers. For instance, Windows XP is version 5.1, Windows 7 is 6.1, and Windows 10 is version 10.0. Understanding these versions is important for system administration and security assessment.

To gather system information, Windows provides powerful tools like Windows Management Instrumentation (WMI), which can be accessed using PowerShell. The `Get-WmiObject` cmdlet allows users to retrieve detailed system data such as OS version, processes, services, and BIOS information. It can also be used for remote system management and administrative tasks.

#### **Important Commands & Information**

`Get-WmiObject` is used to retrieve system information (like services, processes, hardware, and configurations) from Windows using WMI.

```powershell
Get-WmiObject -List
```

To check the operating system version and build number:

```powershell
Get-WmiObject -Class Win32_OperatingSystem | Select Version, BuildNumber
```

To list running processes:

```powershell
Get-WmiObject -Class Win32_Process
```

To list system services:

```powershell
Get-WmiObject -Class Win32_Service
```

To retrieve BIOS information:

```powershell
Get-WmiObject -Class Win32_BIOS
```

To query a remote system:

```powershell
Get-WmiObject -Class Win32_OperatingSystem -ComputerName <Target>
```

The BIOS (Basic Input/Output System) is firmware stored on the motherboard that controls low-level hardware operations such as system startup, hardware initialization, and power management.

Understanding Windows versions, system components, and administrative tools is essential for both system management and cybersecurity assessments, as different versions may have different vulnerabilities and configurations. Further information about the cmdlet can be found [here](https://ss64.com/ps/get-wmiobject.html) and [here](https://adamtheautomator.com/get-wmiobject/).
