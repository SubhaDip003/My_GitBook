# 13. Windows Security

Windows security is an important part of the operating system because it controls how users, applications, and systems access resources. Windows has many built-in features, services, and settings, which makes it powerful but also increases the **attack surface**. Even if a system is fully updated, misconfiguration of these features can still lead to security risks and vulnerabilities.

Over time, Windows has faced many attacks and exploits, both remote and local, because of its wide usage and complex environment. Attackers often target built-in features or weak configurations to gain access or escalate privileges. However, Microsoft has continuously improved Windows security by adding new protections, detection mechanisms, and tools to help administrators secure systems and prevent misuse.

Windows security is based on certain core principles that manage **authentication (who you are)** and **authorization (what you can do)**. These principles apply to users, computers, processes, and threads, ensuring that only permitted actions are allowed. The overall goal of this security model is to reduce unauthorized access and make it harder for attackers or malicious software to compromise the system.

***

## Security Identifier (SID) <a href="#security-identifier-sid" id="security-identifier-sid"></a>

Each of the security principals on the system has a unique security identifier (SID). The system automatically generates SIDs. This means that even if, for example, we have two identical users on the system, Windows can distinguish the two and their rights based on their SIDs. SIDs are string values with different lengths, which are stored in the security database. These SIDs are added to the user's access token to identify all actions that the user is authorized to take.

A SID consists of the Identifier Authority and the Relative ID (RID). In an Active Directory (AD) domain environment, the SID also includes the domain SID.

```powershell
PS C:\htb> whoami /user

USER INFORMATION
----------------

User Name           SID
=================== =============================================
ws01\bob S-1-5-21-674899381-4069889467-2080702030-1002

```

The SID is broken down into this pattern.

```powershell
(SID)-(revision level)-(identifier-authority)-(subauthority1)-(subauthority2)-(etc)

```

Let's break down the SID piece by piece.

| **Number**                      | **Meaning**          | **Description**                                                                                                                                                                                                                  |
| ------------------------------- | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| S                               | SID                  | This just means it is a SID. It tells Windows “this is a Security Identifier”.                                                                                                                                                   |
| 1                               | Revision Level       | This is the version of SID format. It has always been **1**, so you don’t need to worry much about it.                                                                                                                           |
| 5                               | Identifier-authority | <p>This tells which authority created the SID.<br>For example, <strong>5 means NT Authority (Windows system)</strong>.</p>                                                                                                       |
| 21                              | Subauthority1        | This shows that the SID belongs to a **user or group created in a domain or local system**. It basically tells that this SID is for a normal account (not something special like system SID types).                              |
| 674899381-4069889467-2080702030 | Subauthority2        | <p>These numbers together identify the <strong>specific system or domain</strong> where the account was created.<br>👉 Think of this as a <strong>unique ID of the computer/domain</strong>.</p>                                 |
| 1002                            | Subauthority3        | <p>This is the most important part. It identifies the <strong>actual user or group</strong> inside that system/domain.<br>For example:</p><ul><li>500 → Administrator</li><li>501 → Guest</li><li>1000+ → Normal users</li></ul> |
